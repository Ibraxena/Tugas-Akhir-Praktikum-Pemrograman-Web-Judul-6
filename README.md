Weather Dashboard ini merupakan aplikasi web interaktif yang dikembangkan sebagai tugas akhir dalam materi AJAX dan Web Service. Aplikasi ini dirancang untuk menampilkan informasi cuaca secara real-time melalui pemanggilan API dari layanan OpenWeatherMap, dan semua proses pengambilan data dilakukan menggunakan AJAX sehingga halaman dapat berubah secara dinamis tanpa perlu direfresh secara manual oleh pengguna. Aplikasi ini memadukan pemrograman asynchronous menggunakan async/await dengan pemanggilan REST API, kemudian menyajikan hasilnya dalam antarmuka yang modern, responsif, serta mendukung pengaturan tema terang dan gelap. Di dalam aplikasi, pengguna dapat mencari nama kota untuk mendapatkan informasi cuaca terkini, termasuk suhu, kondisi awan, tingkat kelembaban, kecepatan angin, dan suhu terasa (feels like). Selain itu, aplikasi juga menampilkan prakiraan cuaca selama lima hari ke depan yang diolah dari endpoint /forecast. Proses peringkasan data prakiraan ini dilakukan dengan menyeleksi beberapa titik waktu dalam satu hari, sehingga tampilannya konsisten dan mudah dipahami.

Seluruh fungsi tersebut diintegrasikan langsung dalam satu file halaman web yang memuat struktur HTML, styling menggunakan TailwindCSS, serta JavaScript yang menangani logika pemanggilan API, penanganan error, tampilan loader, hingga penyimpanan preferensi pengguna. Aplikasi juga memanfaatkan localStorage untuk menyimpan beberapa pengaturan, seperti tema yang terakhir digunakan, satuan suhu yang dipilih (°C atau °F), daftar kota favorit, dan riwayat kota yang pernah dicari sebelumnya. Penyimpanan lokal ini memastikan setiap pengguna mendapatkan pengalaman yang konsisten meskipun halaman ditutup dan dibuka kembali. Mekanisme auto-refresh juga ditambahkan sehingga data cuaca yang sedang dilihat akan diperbarui secara otomatis setiap lima menit tanpa interaksi pengguna.

Sebagai bagian dari proses pengujian, dua tangkapan layar diambil untuk memperlihatkan bagaimana aplikasi bekerja dalam dua skenario berbeda. Pada gambar pertama, aplikasi berada dalam mode gelap dan menampilkan hasil pencarian untuk Kota Bandung. Pada kondisi ini terlihat bahwa seluruh elemen antarmuka menyesuaikan diri dengan tema gelap, termasuk kartu cuaca, tombol, dan teks. Informasi cuaca terkini ditampilkan dalam satuan Fahrenheit, seperti suhu 78°F dan kondisi “Awan Mendung”. Selain itu, aplikasi juga menampilkan kelembaban, kecepatan angin dalam mph, serta nilai “feels like”. Prakiraan lima hari berikutnya turut muncul dalam bentuk kartu yang menampilkan ikon cuaca, rentang suhu maksimum dan minimum, serta deskripsi singkat. Tampilan ini menunjukkan bahwa mode gelap berfungsi dengan baik, konversi suhu ke satuan Fahrenheit berjalan sebagaimana mestinya, dan semua elemen yang terkait cuaca dapat dirender tanpa kesalahan.

Sementara itu, gambar kedua menunjukkan aplikasi dalam mode terang dengan hasil pencarian untuk Tangerang Selatan. Berbeda dengan gambar pertama, tampilan ini menggunakan satuan Celsius, terlihat dari nilai suhu 27°C dan format kecepatan angin yang menggunakan meter per detik. Tema terang membuat seluruh komponen antarmuka memiliki nuansa cerah namun tetap konsisten dengan desain modern yang digunakan. Daftar kota favorit masih menampilkan Tangerang Selatan sebagai kota yang disimpan, menandakan bahwa fitur penyimpanan favorit melalui localStorage bekerja dengan benar. Prakiraan lima hari juga muncul dengan format yang sama seperti pada mode gelap, sehingga dapat disimpulkan bahwa perubahan tema tidak mempengaruhi stabilitas tampilan maupun struktur data yang dirender.

Kedua hasil pengujian tersebut memperlihatkan bahwa aplikasi berjalan stabil di berbagai kondisi, baik dalam mode terang maupun gelap, serta mampu menangani perbedaan satuan suhu tanpa memengaruhi proses pemanggilan data dari API.
